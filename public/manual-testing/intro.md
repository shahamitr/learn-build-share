# Introduction to Manual Testing

Manual testing is the process of manually testing software for defects. It requires a tester to play the role of an end user and use most of all features of the application to ensure correct behavior.

## Key Concepts

- **Test Case**: A set of conditions or variables under which a tester will determine whether a system under test satisfies requirements or works correctly.
- **Bug/Defect**: An error, flaw, failure or fault in a computer program or system that causes it to produce an incorrect or unexpected result.
- **Regression Testing**: Testing existing software to ensure that a change or addition hasn't broken any existing functionality.

## Why Manual Testing?

Even with the rise of automated testing, manual testing remains crucial for:
- Exploratory testing
- Usability testing
- Ad-hoc testing
- UI/UX validation
