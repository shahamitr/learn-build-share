# 06. Introduction to Pull Requests

## What is a Pull Request (PR)?

A Pull Request is a way to propose changes you've made on a branch to be merged into another branch (usually `main`). It's the heart of collaboration on GitHub.

### Why PRs are Used

*   **Discussion:** Team members can discuss the implementation details.
*   **Review:** Catch bugs, style issues, or security flaws before they hit production.
*   **Documentation:** The PR description serves as a record of *why* a change was made.

### Basic PR Workflow

1.  Create a branch.
2.  Make changes and commit.
3.  Push the branch to GitHub.
4.  Open a Pull Request.
5.  Discuss and review.
6.  Merge.
