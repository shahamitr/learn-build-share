# 09. Markdown for GitHub

## Writing on GitHub

GitHub uses GitHub Flavored Markdown (GFM) everywhere: READMEs, issues, PRs, and comments.

### Formatting Basics

*   **Headers:** `# H1`, `## H2`, `### H3`
*   **Bold:** `**bold**`
*   **Italic:** `*italic*`
*   **Lists:** `- item` or `1. item`

### Code Blocks

Use triple backticks for code.

```javascript
console.log("Hello World");
```

### Tables

| Name | Role |
| :--- | :--- |
| Alice | Dev |
| Bob | QA |

### Task Lists

- [x] Task 1
- [ ] Task 2

## Lab: Improve README Documentation

1.  **Edit:** Open your `README.md` on GitHub (click the pencil icon).
2.  **Add Content:**
    ```markdown
    # Project Title

    ## Features
    - Feature 1
    - Feature 2

    ## Installation
    ```bash
    npm install
    ```
    ```
3.  **Preview:** Click the **Preview** tab to see how it looks.
4.  **Commit:** Commit the changes directly to `main` (for this lab only).
