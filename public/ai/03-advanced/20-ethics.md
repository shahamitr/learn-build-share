# 20. AI and Engineering Ethics

## The Responsibility of the Engineer

As AI becomes more powerful, ethical considerations become paramount. We must ensure our use of AI is fair, safe, and responsible.

### Key Ethical Issues

1.  **Bias:** AI models can reflect biases present in their training data (e.g., gender, race).
2.  **Security:** Generating code that is vulnerable or leaks data.
3.  **Over-Reliance:** Becoming dependent on AI and losing critical thinking skills ("Skill Degradation").
4.  **Intellectual Property:** Using code generated from copyrighted sources without attribution.

### The Engineer's Oath

*   **Verify:** Never trust AI blindly.
*   **Understand:** Don't commit code you don't understand.
*   **Secure:** Prioritize user data privacy.

## Discussion Lab: Ethical Scenarios

### Scenario 1: The "Black Box" Algorithm

You use AI to generate a credit scoring algorithm. It works well, but you don't understand *why* it rejects certain applicants.
*   **Ethical Issue:** Lack of explainability/transparency. Potential for discrimination.
*   **Action:** Audit the logic. Ensure fairness. If you can't explain it, don't ship it.

### Scenario 2: The "Copied" Code

AI generates a complex function that looks identical to a famous open-source library (GPL licensed).
*   **Ethical Issue:** License violation.
*   **Action:** Check for similarity. Use a different implementation or attribute correctly if compatible.
