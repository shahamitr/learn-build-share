# Effective Bug Reporting

A bug report is a document that describes a software defect in detail. A well-written bug report helps developers understand, reproduce, and fix the issue quickly.

## Anatomy of a Bug Report

1. **Bug ID:** A unique identifier.
2. **Title:** A concise description of the issue.
3. **Severity:** How critical the bug is (e.g., Critical, Major, Minor).
4. **Priority:** How soon the bug needs to be fixed (e.g., High, Medium, Low).
5. **Environment:** OS, browser, version, etc.
6. **Pre-conditions:** What needs to be in place before the bug occurs.
7. **Steps to Reproduce:** Clear, step-by-step instructions.
8. **Expected Result:** What should have happened.
9. **Actual Result:** What actually happened.
10. **Attachments:** Screenshots, logs, videos, etc.

## Best Practices

- **Be Specific:** Provide as much detail as possible.
- **One Bug per Report:** Don't combine multiple issues.
- **Reproducible:** Ensure the bug can be reproduced by others.
- **Professional Tone:** Avoid blame or sarcasm.
