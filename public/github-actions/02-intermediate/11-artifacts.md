# 11. Workflow Artifacts

## What are Artifacts?

Artifacts are files generated by your workflow (e.g., binaries, test reports, build folders) that you want to keep after the job finishes.

## Uploading Artifacts

Use the `actions/upload-artifact` action.

```yaml
- name: Upload Build Artifact
  uses: actions/upload-artifact@v4
  with:
    name: build-assets
    path: dist/
```

## Downloading Artifacts

Use the `actions/download-artifact` action to use artifacts in a different job or download them manually from the UI.

## Lab: Upload Build Output

1.  **Modify Build Workflow:** Add the upload step to your build workflow from the previous lab.
    ```yaml
    - run: npm run build
    - uses: actions/upload-artifact@v4
      with:
        name: production-files
        path: dist/  # Or build/ depending on your project
    ```
2.  **Run:** Push the change.
3.  **Verify:** After the run finishes, scroll to the bottom of the Summary page. You should see "production-files" listed under Artifacts. Download and inspect it.
