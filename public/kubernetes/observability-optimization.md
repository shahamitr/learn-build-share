# Observability & Optimization

In this module, we'll explore the more advanced features of Kubernetes observability and optimization.

## Observability
- **Monitoring:**
    - **Prometheus:** A popular monitoring system for Kubernetes.
    - **Visualization:**
        - **Grafana:** A popular visualization tool for Kubernetes.
- **Logging (ELK stack basics):** A popular logging stack for Kubernetes.

## Advanced Deployments
- **Blue-Green deployments:** A deployment strategy that uses two identical production environments.
- **Canary releases:** A deployment strategy that releases a new version of your application to a small group of users first.
- **Multi-cluster setups:** A way to manage multiple clusters in a single environment.

## Performance & Optimization
- **Resource limits/requests:** Specifying the amount of resources a pod needs and can use.
- **Cluster autoscaling:** Automatically scaling the number of nodes in a cluster based on resource availability.
- **Cost optimization:** How to reduce the cost of running a cluster.

## Practical Tip
When optimizing your cluster, always start with the most resource-intensive pods and use resource limits/requests where possible.
