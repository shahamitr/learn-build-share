# 11. Forking and Contributing to Open Source

## Forking

A fork is a copy of a repository that you manage. Forking lets you make changes to a project without affecting the original repository.

### Upstream Repositories

When you fork a repo, the original repo is called the "upstream" repository.

### Contributing Changes

1.  **Fork:** Create a fork of the upstream repo.
2.  **Clone:** Clone your fork locally.
3.  **Branch:** Create a feature branch.
4.  **Commit:** Make changes.
5.  **Push:** Push to your fork.
6.  **PR:** Open a Pull Request from your fork to the upstream repo.

## Lab: Fork Project and Submit PR

1.  **Find Repo:** Go to `https://github.com/octocat/Spoon-Knife`.
2.  **Fork:** Click the **Fork** button in the top right.
3.  **Clone:** Clone your fork locally.
4.  **Edit:** Create a file `CONTRIBUTORS.md` and add your name.
5.  **Push:** Push the changes to your fork.
6.  **PR:** Go to your fork on GitHub and click **Compare & pull request**.
