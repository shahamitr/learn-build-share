# 22. Continuous Integration with GitHub Actions

## Running Tests Automatically

The most common use case for Actions is running tests on every push.

### Build Pipelines

A typical pipeline:
1.  Checkout code.
2.  Install dependencies (`npm install`).
3.  Run linter (`npm run lint`).
4.  Run tests (`npm test`).
5.  Build (`npm run build`).

## Lab: Run Automated Tests

1.  **Create Workflow:** Create `.github/workflows/ci.yml`.
2.  **Content:**
    ```yaml
    name: Node.js CI
    on: [push, pull_request]
    jobs:
      build:
        runs-on: ubuntu-latest
        strategy:
          matrix:
            node-version: [14.x, 16.x, 18.x]
        steps:
        - uses: actions/checkout@v3
        - name: Use Node.js ${{ matrix.node-version }}
          uses: actions/setup-node@v3
          with:
            node-version: ${{ matrix.node-version }}
        - run: npm ci
        - run: npm test
    ```
3.  **Commit:** Push this to a Node.js project (or create a dummy `package.json` with a test script).
4.  **Check:** See the matrix build running 3 jobs in parallel.
