# 08. Issues and Bug Tracking

## Managing Work with Issues

GitHub Issues are used to track ideas, feedback, tasks, or bugs for work on GitHub.

### Issue Anatomy

*   **Title:** Concise summary.
*   **Description:** Detailed explanation.
*   **Labels:** Categorization (e.g., `bug`, `enhancement`, `documentation`).
*   **Assignees:** Who is responsible for this issue.

### Issue Templates

Maintainers can create templates to ensure users provide necessary information (like reproduction steps for bugs).

## Lab: Create Bug Report Issue

1.  **Issues Tab:** Go to the **Issues** tab in your repository.
2.  **New Issue:** Click **New issue**.
3.  **Details:**
    *   **Title:** Fix typo in README
    *   **Comment:** "There is a spelling mistake in the introduction section."
4.  **Sidebar:**
    *   **Assignees:** Select yourself.
    *   **Labels:** Select `bug` and `good first issue`.
5.  **Submit:** Click **Submit new issue**.
6.  **Reference:** Note the issue number (e.g., #1). In a future commit, you can close this issue by typing `Fixes #1`.
