# Pipeline Structure & Variables

In this module, we'll dive deeper into the structure of GitLab CI/CD pipelines and the use of variables.

## Advanced Pipeline Structure
- **Multiple Stages:** Defining the order of execution for your jobs (e.g., build → test → deploy).
- **Job Dependencies:** Specifying that a job depends on the output of another job.
- **Artifacts & Caching:** 
    - **Artifacts:** Files generated by a job that can be used by subsequent jobs.
    - **Caching:** Reusing files between jobs to speed up the pipeline.
- **YAML Anchors (Reusability):** Reusing YAML snippets across multiple jobs to keep your `.gitlab-ci.yml` file clean and DRY (Don't Repeat Yourself).
```yaml
.test_template: &test_definition
  stage: test
  script:
    - echo "Running tests..."

unit-test:
  <<: *test_definition
  script:
    - echo "Running unit tests specifically"

integration-test:
  <<: *test_definition
  script:
    - echo "Running integration tests specifically"
```

## Working with Variables
Variables are used to store and reuse information in your pipeline.
- **Predefined Variables:** Provided by GitLab (e.g., `$CI_COMMIT_REF_NAME`, `$CI_PROJECT_ID`).
- **Custom Variables:** Defined in `.gitlab-ci.yml` or in the GitLab UI (Settings > CI/CD > Variables).
- **Variable Types:**
    - **Variables:** Standard key-value pairs.
    - **File Variables:** Store the content of a file.
- **Variable Security:**
    - **Protected Variables:** Only available on protected branches or tags.
    - **Masked Variables:** Hidden in job logs. (Ideal for API keys and passwords)

## Practical Tip
When using variables, always be careful with sensitive information (e.g., API keys, passwords) and use masked variables if possible.
