# Integration, System & Exploratory Testing

In this module, we'll explore more advanced testing types and techniques.

## Advanced Testing Types
- **Integration Testing:** Verifies that different modules or services work together as expected.
- **System Testing:** Testing the entire system as a whole to ensure it meets requirements.
- **UAT (User Acceptance Testing):** Testing performed by the end-user or client to verify that the system meets their needs.
- **Exploratory Testing:** Testing without predefined test cases, using heuristics and intuition.
- **Ad-hoc Testing:** Testing without any formal documentation or plan.

## Exploratory Testing
Exploratory testing is a powerful technique for finding bugs that are often missed by scripted test cases. It involves:
- **Learning:** Understanding the software's behavior.
- **Test Design:** Creating tests on the fly.
- **Test Execution:** Running the tests.
- **Reporting:** Documenting findings.

## Practical Tip
When performing exploratory testing, focus on edge cases and unexpected user behaviors.
