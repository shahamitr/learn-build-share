# Exploratory Testing

Exploratory testing is a powerful manual testing technique where the tester explores the software without predefined test cases.

## Key Concepts

- **Learning while Testing:** The tester learns about the software's behavior as they test it.
- **Heuristics:** Using "rules of thumb" or mental models to guide testing.
- **Creativity:** Thinking outside the box to find edge cases.
- **Agility:** Adapting to new information and findings.

## Best Practices

- **Time-Boxed Sessions:** Set a specific time limit for each session.
- **Chartered Testing:** Define a clear goal or "charter" for each session.
- **Document Findings:** Record what was tested, what was found, and any questions.
- **Debriefing:** Share findings with the team after each session.
