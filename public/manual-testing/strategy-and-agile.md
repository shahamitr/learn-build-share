# Test Strategy & Agile QA

In this module, we'll explore the strategic and collaborative aspects of software testing in modern development environments.

## Test Strategy & Planning
- **Test Strategy:** A high-level document that defines the overall testing approach for an organization or a large project.
- **Test Plan:** A detailed document that describes the testing objectives, scope, resources, schedule, and deliverables for a specific project.
- **Risk-Based Testing:** Prioritizing testing efforts based on the likelihood and impact of potential failures.
- **Estimation Techniques:**
    - **WBS (Work Breakdown Structure):** Breaking down the project into smaller tasks.
    - **Three-Point Estimation:** Optimistic, Pessimistic, and Most Likely estimates.
    - **Delphi Technique:** Expert-based estimation.

## QA in Agile (Scrum)
Agile is an iterative and incremental approach to software development.
- **Scrum Framework:** Roles (Product Owner, Scrum Master, Team), Events (Sprint, Sprint Planning, Daily Standup, Sprint Review, Sprint Retrospective), and Artifacts (Product Backlog, Sprint Backlog, Increment).
- **Role of QA in Agile:**
    - **Shift Left Testing:** Starting testing activities as early as possible in the SDLC.
    - **Continuous Feedback:** Providing feedback to developers throughout the sprint.
    - **Collaborative Testing:** Working closely with developers and product owners.
- **User Stories & Acceptance Criteria:**
    - **User Story:** A short, simple description of a feature from the user's perspective.
    - **Acceptance Criteria:** The conditions that a user story must satisfy to be considered complete.
- **Definition of Done (DoD):** A shared understanding of what it means for a task to be complete (including testing).

## Modern QA Practices
- **TDD (Test-Driven Development):** Writing tests before writing code.
- **BDD (Behavior-Driven Development):** Using natural language to describe the behavior of the application (Given-When-Then).
- **Exploratory Testing in Agile:** Using exploratory testing to find bugs that automated tests might miss.

## Practical Tip
In Agile, **QA is a team effort**. Everyone is responsible for quality. Participate actively in all Scrum ceremonies and advocate for quality at every stage.
