# Web, API & SQL for Testers

In this module, we'll gain hands-on experience with web UI, APIs, and SQL.

## Web Application Testing
- **Client-server Architecture Basics:** Understanding how web applications work.
- **Browser Compatibility Testing:** Testing that the application works as expected across different browsers.
- **UI/UX Validation:** Verifying that the user interface is intuitive and easy to use.

## API Testing (Manual)
- **Basics of APIs:** Understanding what APIs are and how they work.
- **HTTP Methods:** GET, POST, PUT, DELETE.
- **Status Codes:** 200 (OK), 404 (Not Found), 500 (Internal Server Error).
- **Postman:** A popular tool for testing APIs.

## SQL for Testers
- **Basic Queries:** SELECT, INSERT, UPDATE, DELETE.
- **Joins:** INNER JOIN, LEFT JOIN.
- **Validating Backend Data:** Ensuring that data is correctly stored and retrieved from the database.

## Practical Tip
When testing APIs, always verify the response status code and the response body.
