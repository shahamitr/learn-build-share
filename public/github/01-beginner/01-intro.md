# 01. Introduction to GitHub

## What is GitHub?

GitHub is a cloud-based platform that hosts Git repositories. While Git is the tool that tracks changes in your code, GitHub is the website where you store that code and collaborate with others.

### Git vs GitHub

| Feature | Git | GitHub |
| :--- | :--- | :--- |
| **Type** | Version Control System (VCS) | Hosting Service for Git Repositories |
| **Location** | Runs locally on your computer | Runs in the cloud (web-based) |
| **Usage** | Tracking changes, branching, merging | Collaboration, code review, issue tracking |
| **Offline?** | Yes, works without internet | No, requires internet connection |

### Why Teams Use GitHub

1.  **Collaboration:** Multiple developers can work on the same project simultaneously.
2.  **Backup:** Your code is safely stored in the cloud.
3.  **Code Review:** Tools like Pull Requests allow teams to review code before merging.
4.  **Project Management:** Issues and Projects help track bugs and features.
5.  **CI/CD:** GitHub Actions automates testing and deployment.

## Lab: Create Your First GitHub Repository

1.  **Sign Up:** Go to [github.com](https://github.com) and create an account if you haven't already.
2.  **Create Repo:** Click the **+** icon in the top right corner and select **New repository**.
3.  **Name It:** Enter `my-first-repo`.
4.  **Settings:**
    *   Select **Public**.
    *   Check **Add a README file**.
5.  **Create:** Click **Create repository**.

Congratulations! You now have a repository hosted on GitHub.
