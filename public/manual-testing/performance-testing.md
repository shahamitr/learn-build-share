# Performance Testing Basics

Performance testing is a non-functional testing technique performed to determine the system parameters in terms of responsiveness and stability under various workload conditions.

Performance testing measures the quality attributes of the system, such as scalability, reliability, and resource usage.

## Why Performance Testing?

*   **Determine Speed:** Does the application respond quickly enough?
*   **Determine Scalability:** Can the application handle a large number of users?
*   **Determine Stability:** Does the application remain stable under heavy loads?
*   **Identify Bottlenecks:** Find the root cause of performance issues.

## Types of Performance Testing

1.  **Load Testing:** Checks the application's ability to perform under anticipated user loads. The objective is to identify performance bottlenecks before the software application goes live.
2.  **Stress Testing:** Involves testing an application under extreme workloads to see how it handles high traffic or data processing. The objective is to identify the breaking point of an application.
3.  **Endurance Testing:** Done to make sure the software can handle the expected load over a long period of time.
4.  **Spike Testing:** Tests the software's reaction to sudden large spikes in the load generated by users.
5.  **Volume Testing:** Under Volume Testing large no. of. Data is populated in a database and the overall software system's behavior is monitored.
6.  **Scalability Testing:** The objective of scalability testing is to determine the software application's effectiveness in "scaling up" to support an increase in user load.

## Common Performance Metrics

*   **Response Time:** The time from when a user enters a request until the first character of the response is received.
*   **Throughput:** The number of requests processed per unit of time (e.g., requests per second).
*   **Error Rate:** The percentage of requests resulting in errors compared to all requests.
*   **CPU Utilization:** The amount of time the CPU spends processing instructions.
*   **Memory Utilization:** The amount of physical memory available to processes on a computer.

## Tools for Performance Testing

*   **JMeter:** A popular open-source tool for load testing and performance measurement.
*   **LoadRunner:** A widely used performance testing tool from Micro Focus.
*   **Gatling:** An open-source load and performance testing framework based on Scala, Akka, and Netty.
*   **k6:** A modern load testing tool, using JavaScript to write test scenarios.
