# Writing Effective Test Cases

Test cases are the building blocks of manual testing. A well-written test case ensures that testing is consistent, repeatable, and easy to understand.

## Anatomy of a Test Case

1. **Test Case ID:** A unique identifier.
2. **Title:** A brief description of what is being tested.
3. **Pre-conditions:** What needs to be in place before the test starts.
4. **Test Steps:** Step-by-step instructions.
5. **Test Data:** Specific inputs required.
6. **Expected Result:** What should happen if the software works correctly.
7. **Actual Result:** What actually happened (filled during execution).
8. **Status:** Pass/Fail/Blocked.

## Best Practices

- **Be Clear and Concise:** Avoid ambiguity.
- **One Goal per Test Case:** Don't try to test everything at once.
- **Independence:** Test cases should ideally not depend on each other.
- **Traceability:** Link test cases to requirements.
