# CI/CD & Git Foundations

Welcome to the world of DevOps! This module covers the core concepts of CI/CD and the version control system that powers it.

## What is CI/CD?
CI/CD is a method to frequently deliver apps to customers by introducing automation into the stages of app development.
- **Continuous Integration (CI):** The practice of automating the integration of code changes from multiple contributors into a single software project.
- **Continuous Delivery (CD):** Automated delivery of code to a repository after which it can be manually deployed to production.
- **Continuous Deployment (CD):** Automated deployment of code to production directly after passing all tests.

## Core Concepts
- **Pipeline:** A top-level component of CI/CD that contains your jobs and stages.
- **Stages:** Define *when* to run jobs (e.g., build, test, deploy).
- **Jobs:** Define *what* to do (e.g., compile code, run tests).
- **Artifacts:** Files generated by a job that can be used by subsequent jobs or downloaded.
- **Caching:** Speeding up pipelines by reusing dependencies from previous runs.

## Git Basics (Essential)
- **Git:** A distributed version control system.
- **Repository:** A central location where your project's files and history are stored.
- **Commit:** A snapshot of your changes.
- **Branch:** An independent line of development.
- **Merge vs Rebase:** 
    - **Merge:** Combines two branches, creating a new "merge commit".
    - **Rebase:** Re-applies commits on top of another base tip, creating a linear history.
- **Pull/Merge Requests:** A way to propose changes and have them reviewed before merging into the main branch.
- **Git Workflow:**
    1. `git clone`: Copy a repository to your local machine.
    2. `git checkout -b <branch>`: Create and switch to a new branch.
    3. `git add .`: Stage your changes.
    4. `git commit -m "message"`: Commit your changes.
    5. `git push origin <branch>`: Push your changes to the remote repository.
    6. Create a Merge Request (MR) in GitLab.

## The DevOps Lifecycle
- **Plan:** Defining the project's goals and requirements.
- **Code:** Writing and reviewing the application's source code.
- **Build:** Compiling and packaging the application.
- **Test:** Running automated tests to ensure quality.
- **Release:** Preparing the application for deployment.
- **Deploy:** Deploying the application to production.
- **Operate:** Monitoring and managing the application in production.
- **Monitor:** Collecting and analyzing data to improve the application.
