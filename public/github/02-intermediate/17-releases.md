# 17. Managing Releases

## Creating Releases

Releases are deployable software iterations. They are based on Git tags.

### Semantic Versioning

Use SemVer (e.g., v1.0.0, v1.1.0, v2.0.0) to communicate the impact of changes.

### Release Notes

GitHub can automatically generate release notes based on merged PR titles.

## Lab: Publish a Release

1.  **Code Tab:** Go to the **Code** tab.
2.  **Releases:** Click **Releases** in the sidebar.
3.  **Draft a new release:** Click the button.
4.  **Tag version:** `v1.0.0`.
5.  **Target:** `main`.
6.  **Title:** "Initial Release".
7.  **Description:** Click **Generate release notes**.
8.  **Publish:** Click **Publish release**.
