# 07. Basic Code Reviews

## Reviewing PRs

Code review is a critical skill. It's not just about finding bugs; it's about knowledge sharing and maintaining code quality.

### Adding Comments

*   **General Comments:** Comments on the PR as a whole.
*   **Inline Comments:** Comments on specific lines of code. Hover over a line number in the "Files changed" tab and click the `+` icon.

### Review Status

*   **Comment:** General feedback without explicit approval or rejection.
*   **Approve:** The changes look good and are ready to merge.
*   **Request Changes:** There are issues that must be addressed before merging.

## Lab: Review a PR (Self-Review)

Since you are working alone, you will review your own PR.

1.  **Go to PR:** Open the PR you created in Module 06.
2.  **Files Changed:** Click the **Files changed** tab.
3.  **Comment:** Hover over the line you added. Click `+`. Type: "This looks correct." and click **Start a review**.
4.  **Finish Review:** Click **Finish your review** (top right). Select **Comment** (you can't approve your own PR usually). Click **Submit review**.
5.  **Merge:** Now, go back to the **Conversation** tab and click **Merge pull request**, then **Confirm merge**.
6.  **Delete Branch:** Click **Delete branch** to clean up.
